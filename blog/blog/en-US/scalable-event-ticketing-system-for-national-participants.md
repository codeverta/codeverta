# Building a Scalable Event Ticketing System for Hundreds of National Participants Without Downtime

Imagine organizing a trail running event at Mount Manglayang, Bandung. Participants come from Aceh, Papua, Kalimantan, Sulawesi, and many other regions. Hundreds of people want to register, pay, and receive electronic tickets at the same time.

During peak registration, a weak system can fail: payments get stuck, confirmation emails arrive late, and participants complain in WhatsApp groups.

This is the type of problem Codeverta helped Manglayang Academia solve.

## Common Problems in Large Event Registration

Event organizers often start with manual forms or generic tools. That works for small events, but becomes risky when demand grows.

Common issues include:

- Sudden traffic spikes after Instagram announcements
- Messy payment confirmation
- Duplicate participant data
- Slow manual verification
- Long check-in queues on race day
- Poor experience for participants with unstable internet

For trail running events, reputation matters. One technical problem can quickly spread through the community.

## Codeverta's Approach

We did not build a generic landing page. We mapped the real user journey with the organizer:

1. Participant opens the event page.
2. Participant chooses a category.
3. Participant fills personal data.
4. Participant uploads qualification proof if needed.
5. Participant completes payment.
6. Admin monitors registration and verifies data.
7. Participant receives ticket and confirmation.

## Key System Features

- Mobile-first registration page
- Category selection
- Participant data form
- Payment integration
- Real-time admin dashboard
- Public entry list
- Exportable participant data
- QR ticket support
- Monitoring and fallback preparation

## Result

The new web app helped participants register smoothly from different regions. Admins could monitor payment status and participant data from one dashboard. The public entry list increased transparency for the community.

Most importantly, the system remained stable during traffic spikes.

## Lessons for Event Organizers

- Do not wait until registration day to test traffic
- Keep the registration flow short
- Make payment confirmation fast
- Give admins real-time visibility
- Prepare QR check-in for race day
- Design for mobile users first

## Conclusion

A serious event needs more than a poster and a form. It needs a reliable registration and ticketing system.

For events with hundreds or thousands of participants, scalability, payment reliability, and operational visibility are essential.
